# builder 0.6.1

## Highlights

- The exporter no longer emits the retired `allowAgentSwap` / `allowPromptOverride` fields on an
  AI Step or AI Debate step's `retryPolicy`. Both fields were always decorative — the runtime
  never read them — and the framework's `RetryPolicy` schema definition was tightened to a
  three-field shape (`allowRetry`, `allowRetryFromPrevious`, `maxAttempts`) with
  `additionalProperties: false`, rejecting any document that still declares either field. Every
  workflow this version exports validates cleanly against the current schema.

## Breaking changes

None. This is a patch-level export-shape correction: the two removed fields never had a
supported public meaning, no `WorkflowBuilderProps` or exported type changes.

## Compatibility

Exports and validates workflow documents at `schemaVersion` 1
(`agentforge4j.supportedSchemaVersions` in `package.json`, unchanged from 0.6.0). Publishes to
the npm dist-tag `latest`.

## Coordinates
