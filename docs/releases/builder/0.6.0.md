# builder 0.6.0

## Highlights

- Draft-recovery persistence: the canvas model is now saved automatically (debounced) as it is
  edited and silently restored on the next load, with a dismissible "Restored your previous
  session" notice offering "Start fresh". Falls back to a built-in `localStorage`-backed
  implementation with no server required; host applications can supply their own
  `persistence: { load, save, clear }` adapter on `WorkflowBuilderProps` to save drafts to their
  own backend instead. Restore-on-mount is suppressed when a host already seeds real
  `initialWorkflow` content or in read-only mode, so a stale local draft can never clobber
  host-provided data.
- Full undo/redo across every meaningful change (add/delete/configure/connect/reorder/metadata/
  start-step), not just delete — Undo/Redo toolbar buttons plus Ctrl+Z / Ctrl+Shift+Z / Ctrl+Y
  (suppressed while a text field has focus). A shared confirmation dialog now gates step
  deletion from both the inspector and the canvas Delete/Backspace key. Also fixes a latent bug
  where dragging an existing edge's endpoint to reroute it silently snapped back on release.
- Validation feedback is now actually reachable: the validation summary popover is portaled to
  `document.body` (it previously rendered invisibly behind an open inspector panel, a stacking
  context no z-index change could escape), and the guided checklist's "Require approval" action
  now selects the step, opens the Behavior section, and focuses the approval field instead of
  silently mutating the step's transition with no visible feedback.
- Start-step indicator/chooser generalized beyond a single-node call-to-action, and visible,
  persistent confirmation (filename) after a successful Export.
- Step-library panel scrolling fixed — all 6 step types (previously 3 of 6 silently clipped with
  no scroll affordance) are reachable at desktop/tablet widths.
- Below a 767px measured container width, the builder now renders a desktop-required notice
  instead of the full editor, replacing an unusable partial-editor experience (dead "+ Add step"
  control, obstructed name field, canvas-covering overlay) with an honest, deliberate boundary.
  The gate measures the builder's own rendered container (`ResizeObserver`, border-box), not the
  browser viewport, so it also applies correctly inside a narrow host panel.

## Breaking changes

No breaking public API changes. Version 0.6.0 adds new optional persistence APIs
(`WorkflowBuilderProps.persistence`, the exported `BuilderPersistenceAdapter` type) and widens
`BuilderAdapters.exportBundle`'s return type to `Promise<void | ExportOutcome>` so an adapter may
optionally report back the produced filename — both additive, source- and binary-compatible with
0.5.0. The one user-visible behaviour change worth calling out explicitly: below the 767px
container-width breakpoint, the builder now renders a desktop-required notice instead of
attempting to render the editor — a host previously relying on the editor being present (however
unusably) at narrow widths will now see the notice instead.

## Compatibility

Exports and validates workflow documents at `schemaVersion` 1
(`agentforge4j.supportedSchemaVersions` in `package.json`, unchanged from 0.5.0). Publishes to
the npm dist-tag `latest`.
