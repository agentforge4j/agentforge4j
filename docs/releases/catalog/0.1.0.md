# catalog 0.1.0

The first public release of the shipped workflow and agent catalog: the built-in workflows
(and their bundled agents) that a framework consumer gets simply by adding this module to the
classpath, with no code required to wire them in.

## Highlights

- **Workflow Execution Estimator** — analyzes a workflow definition and produces a full
  estimate (shape, turn counts, and a token-range envelope with a recommendation and
  confidence) before it runs, disclosing the complete estimate in-workflow ahead of a human
  approval pause.
- **Agent Creator** — turns a freeform agent idea into a downloadable, deterministically-
  validated agent bundle (definition, system prompt, and behavioural boundaries) behind a
  human approval gate.
- **Compatibility manifest** — every shipped workflow is verified in-reactor against the real
  catalog (conformance, data-driven scenario execution, scenario-schema conformance, manifest
  compatibility), and the catalog fails fast under an incompatible or missing framework rather
  than loading partially.

## Breaking changes

None. This is the first public release.

## Compatibility

This release's manifest declares `minimumAgentForge4jVersion` and `maximumAgentForge4jVersion`
both pinned to `0.1.0` — **catalog 0.1.0 requires framework 0.1.0, exactly** (see
[ADR-0032](../../adr/0032-catalog-compatibility-as-an-exact-pinned-framework-version.md)). A
framework of any other version rejects this catalog at load time
(`CatalogCompatibilityGate`) rather than loading it partially or silently. Declares
`workflowSchemaVersion: 1`, matching the framework 0.1.0 workflow schema.

## Coordinates

```xml
<dependency>
  <groupId>org.agentforge4j</groupId>
  <artifactId>agentforge4j-workflows-catalog</artifactId>
  <version>0.1.0</version>
</dependency>
```

The catalog is not implied by any `${project.version}` inheritance — consumers pin this
version explicitly, independent of the framework version they build against (subject to the
compatibility bound above).

## Support
