# Why AgentForge4j

AgentForge4j is an embeddable Java framework for Java teams building their own governed
AI workflows on top of large language models. Workflows are declared as data — JSON
definitions validated against published schemas — and executed by a runtime that follows
that definition faithfully: explicit steps, human approval and review gates, retries,
structured tool invocations, and a replayable event log for observability.

It is a framework, not a product: you embed it in your own application. There is no
required hosting, account, or service — providers, tools, and file sinks are pluggable
through service-provider interfaces (SPIs), and a Spring Boot starter is available for
applications that use Spring.

## What it gives you

- **Declarative workflows.** Steps, behaviours, and transitions described in JSON and
  validated against the published schemas before they run.
- **Explicit lifecycle control.** Human approval, review, and input gates pause a run for
  a real decision exactly where the workflow author placed one; step-level retries are
  enforced as hard, atomic ceilings, not best-effort.
- **Observability.** Every run emits a structured, replayable event log, so what a
  workflow did — and why — stays inspectable after the fact.
- **Pluggable LLM providers and integrations.** Multiple providers are discovered at
  runtime through a `ServiceLoader` SPI; add or swap providers, tools, and integrations
  without touching workflow definitions.
- **Two integration styles.** A framework-agnostic bootstrap builder, or a Spring Boot
  starter with configuration properties.
